Using Git

Version control = Central repo, 1 file name, full history

along comes git

Distributed vc system

one project folder, hosted on a server, 1 filename, with a history of every change ever made to that file.

very basic using one document, but this scales easily as we'll see soon.

there are some key terms you need to know about using Git

Clone

Copy repo to your local machine

files are on a server, and you need to make a local copy of it in order to make changes.

Similar to Photocopying the master copy

So we need to make some changes to that file

Commit

Make your changes locally

Commit: A snapshot of your project in time.

you make changes to this photocopy, while leaving the original on the server untouched.

all changes thus far are local to you, they are not on the server yet

Once you are done, you'll want to push your changes to the server, appropriately named "push command" »

Push (Sync)

Apply your changes to central repo

push your changes to the server

Your Photocopy has now become the new master copy

old master copy still fully intact in its old state, It's just filed away for easy access if you evry wanted to go back or view the changes( better metaphor)

Let's walk through how to create your first repo on GitHub

Create Your First Repo

Creating a repo is really simple using the GitHub for windows client.

Drag and Drop

Drop an existing folder into the GitHub app.

This animation shows you just how easy it is.

it's literally a drag and drop operation to turn any existing project into a GitHub repo.

We'll create a new repo from a project we've been working on about Kittens

After you drag the folder into GHW, you'll get prompted to fill in some information about it »

Name it

We'll come up with a good name for our repo, we'll call it, Kittens.

Here you can decide if you want to push it to gitHub immediately or not, I'll say no this time, and show you how to do it in the GUI.

It's that simple to create a new project on GitHub, let's take a look at how the program works and see our repo now.

GUI Home Screen

This is the screen you'll be presented with when you start the application.

GitHub Repo List

Clicking on your username under the "GitHub" category will show you any repo that you haven't cloned yet

The list on the right is populated with any repositories that you have created or forked on GitHub

double click a repo in this list to clone that repo onto your local machine.

Once a repo has been cloned locally it will be located here and under the Local category

Local Repo List

When you select "repositories" under the "local" category, the right side of the screen displays a list of any local repos.

To be a local repo, you must have either not synced it to GitHub yet, like our kittens repo, or you have cloned it to your local computer from GitHub.

Let's take a look at our Kittens repo now. WE'll double click on the Kittens repo and we'll be brought to the repo screen.

Committing

So we created this Kittens repo, but we havent pushed it to GitHub yet. Let's do that now.

On the repo screen you'll see a box on the top left that tells us that we havent committed the new repo to the server yet.

This commit box will also be present any time you make any changes to your project

We'll need to provide a short message indicating the changes we made.

Since this is the first time we're pushing this to GitHub I prefer to just make that message "Initial Commit" and click the commit button at the bottom of that box.

Commit List

after makign the commit, we'll notice that it gets added to the top of the commit list.

this list allows you to click on different commits throughout the project's lifespan and see all of the changes that commit featured.

We only see one for now, but each additional commit would get stacked on top of our initial commit

Differences List

When you click each commit, youll see a line by line detailed list of changes associated with that commit on the right side of the screen.

That is called the The differences list or DIFF list

Lines that were changed will show up with a red background and the original code that was on that line

below it you'll see a line of text with a green background detailing the newly added or changed code

You'll be able to see your changes line by line

Let's push this baby to GitHub now!

Making Magic

The top right of the window features the button that makes all the GitHub magic happen.

Press that "Push to GitHub" button and you'll send all your local changes to GitHub

When dealing with repos that have already been pushed to GitHub...

Making Magic

That button icon and title will change to a double arrow button and the text will say Sync.

you'll see a small icon next to the sync button when you have changes that need to be synced to teh server

And that small number icon will disappear when you have no more changes to push to GitHub.

News Feed

First thing you see when you log into github.com is your news feed.

You see activity of the people, repos and issues that you are following

Think of it like a facebook feed, but with project and user information you actually care about instead of pictures of half eaten lunches, or offensive political rants from people you haven't seen in 20 years

Search

The search bar at the top of the page is context aware.

It's search results it displays are relative to where you started the search from.

If you are on a repo page, it will search the repo, but if you are on a user page, or the home page, it will search the whole site.

You can always click the text next to it to widen or narrow the search context.

Each GitHub user has their own user page. For demonstration purposes let's type in my username here

User Page

You can see an overview of any users GitHub activity on their page

You can also follow a user to see what they do in the future as well.

you'll see a list of all of their repos as well, lets click my jquery plugin to see what a repo page looks like

Repo Page

Every GitHub repo has its own page on the site

at the top of every repo page you'll see a short description of that repo if the author created one.

below that you'll see a general activity report of that repo

below that you'll see all the files associated with it.

You'll notice a couple of buttons on this page as well.

Clone & Download

on the right side of the screen you'll see the clone and download buttons.

clicking the clone in desktop button will actually bring up the GUI client and clone the repo onto your local machine.

clicking the download button will download a zip file of the entire repo's content

Watching & Starring Repos

if you watch a repo, any changes that happen on it will show up in your news feed

starring a repo is like favoriting it, and you'll have a list of all your starred repos in one place.
