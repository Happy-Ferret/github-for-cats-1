Using Git

Version control = Central repo, 1 file name, full history

  • Clone: Copy repo to your local machine
  • Commit: Make your changes locally
  • Push/Sync: Apply your changes to central repo

Distributed vc system, on an open source platform.

Similar to Photocopying the master copy, changes made to the photocopy become the new master copy

Commit: A snapshot of your project in time.

My files are now located on a server, and I make local modifications to them and push changes back to the server.

Using GitHub

Version control = Public central repo

  • Fork: Create their own copy of your repo
  • Correct it or add new features
  • Pull: Request to pull their changes into your central
